# GrowthMate 3 分钟 Demo 口播脚本

## 使用方式

建议视频控制在 2 分 50 秒到 3 分钟。口播不要把所有功能都讲完，重点讲清楚“普通 KOC 痛点、Agent 工作流、评测闭环、差异化”。

## 0:00-0:25 开场：痛点和产品定位

画面：打开 GrowthMate 首页。

口播：

大家好，我的作品是 GrowthMate，一个面向普通 KOC 的 AI 内容增长 Agent。

我想解决的问题不是“帮用户写一句文案”，而是普通 KOC 从 0 到 1 起号时更真实的困难：不知道自己该做什么定位，第一周该发什么，发完以后看什么数据，以及下一条内容应该怎么改。

我自己有小红书账号运营经验，也做过评测 PM 相关实习，所以这个产品不只关注生成结果，而是更关注“生成之后能不能执行、能不能复盘、能不能持续迭代”。

## 0:25-1:15 主流程：从画像到 7 天增长任务

画面：进入主应用，填写或展示一个 KOC 画像。

口播：

用户只需要输入一些低压力信息，比如自己是谁、目标用户是谁、想做什么平台、已有素材是什么、当前最大的卡点是什么。

GrowthMate 会先做 Agent 诊断，而不是直接给标题。它会判断用户当前账号阶段、主要阻塞点、可用素材和不建议做的事情。

接着，系统会生成 7 天增长任务。这里的重点不是随机选题，而是一个有阶段递进的路径：前两天先建立信任，中间验证方向，后面再做轻转化或系列延展。

每条任务都会绑定用户已有素材、发布后的观察信号和评论区预埋问题。这样用户不是只拿到一个标题，而是知道今天为什么发这条、发完以后看什么反馈。

## 1:15-1:50 内容生成：分步工作流

画面：点击第一条任务，展示内容生成结果。

口播：

GrowthMate 采用的是分步式工作流，不会一次性生成 7 天所有正文。因为普通创作者真正执行时，通常是先确定一周方向，再选择当前最重要的一条内容来打磨。

用户点进某一条任务后，系统会根据目标平台生成完整内容。如果是小红书，会生成封面标题、正文、配图计划和话题标签；如果是抖音或视频号，会生成口播脚本、镜头建议、字幕重点和评论触发问题。

这一步也会强调用户自己的素材和经历，避免变成泛泛的运营课话术。

## 1:50-2:15 复盘：基于真实发布稿和数据

画面：切到复盘区，展示输入真实发布稿、数据、评论样本。

口播：

一个很重要的设计是，用户大概率不会原样发布 AI 生成稿，他一定会自己改。所以 GrowthMate 的复盘允许用户输入自己真实发布的内容、数据和评论样本。

Agent 会区分“用户真实发布稿”和“AI 建议稿”。真实发布稿是一手证据，AI 建议稿只能作为对比基准。系统会判断问题更可能出在标题、开头、素材、内容结构还是互动设计，并推荐下一条内容任务。

## 2:15-2:50 评测后台：差异化亮点

画面：切到 `/eval`，展示评测集管理、评测运行看板、批次详情。

口播：

我认为这个项目最有差异化的部分是评测后台。

因为这类生成型产品很容易变成“看起来不错，但不知道是不是真的有用”。所以我设计了 5 个固定评测集，覆盖小红书、抖音、视频号、公众号等不同平台，也覆盖起号、互动不稳定、商业化敏感等不同场景。

每次 Agent 生成结果后，会进入 LLM Judge。Judge 会按照 Rubric 给出分数、理由、证据、主要问题和 Prompt 迭代建议。

更重要的是，我做了快照重评机制。也就是说，如果我只是调整 Judge Prompt，不需要重新生成内容，可以直接复用历史 Agent 输出重新评审。这样能区分到底是产品输出变了，还是评测标准变了。

这里有一个真实迭代案例：敏感肌小红书 case 早期被评测发现 note 正文缺失、复盘证据边界不清。修复后，这个 case 的模型评审结果从 B 提升到了 A。

## 2:50-3:00 总结

画面：回到产品或评测看板。

口播：

所以 GrowthMate 的核心不是单纯让 AI 写内容，而是把 AI 放进普通 KOC 的完整增长链路里：诊断、规划、生成、复盘、评测、再迭代。

它结合了真实创作者经验和评测 PM 方法论，希望让普通 KOC 不只是获得一篇文案，而是获得一套可以持续改进的内容增长系统。

## 备用短版口播

如果时间不够，可以用这版：

GrowthMate 是一个面向普通 KOC 的 AI 内容增长 Agent。它解决的不是单点文案生成，而是普通创作者从定位、选题、内容生成到发布后复盘的一整套增长问题。

用户输入自己的平台、赛道、人设、已有素材和当前卡点后，Agent 会先做账号诊断，再生成 7 天增长任务。每条任务都包含目标、素材、观察信号和评论区预埋。用户选择其中一条后，系统再生成当前平台可发布的完整内容。

发布后，用户可以输入真实数据、评论样本和自己实际改过的发布稿。Agent 会基于真实发布稿做归因，并推荐下一条内容任务。

这个项目最大的差异化是评测后台。我设计了固定评测集和 LLM Judge，用来评估 Agent 输出质量。Judge 不只给分，还会给理由、证据、问题和 Prompt 迭代建议。同时支持复用历史输出重评，避免每次改评测标准都重新生成内容。

因此，GrowthMate 不只是生成内容，而是形成了“生成、评测、迭代”的闭环。
