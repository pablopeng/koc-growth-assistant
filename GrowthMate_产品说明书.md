## 模块一：用户洞察与问题定义

### 1. 目标用户

GrowthMate 面向普通 KOC、内容创作新手、低粉但有真实经验积累的个人创作者。典型用户不是专业 MCN 团队，也不是已经有稳定内容方法论的成熟达人，而是有一些真实经历、素材和表达欲，但不知道如何把它们转化为稳定账号增长的人。

典型用户包括：

- 刚准备起号的新手 KOC，不知道自己适合做什么定位。
- 已经发过内容，但数据不稳定、互动少的普通创作者。
- 有真实经历和素材积累，但不知道如何包装成平台内容的人。
- 想接商单、做个人品牌或积累影响力，但缺少系统规划的用户。

### 2. 用户痛点

普通 KOC 的核心问题并不是“不会写一句文案”，而是缺少一套从定位到发布再到复盘的内容增长流程。

具体痛点包括：

- **定位模糊**：用户知道自己想做某个方向，但无法提炼出有差异化、有记忆点的账号切口。
- **选题低效**：每天靠刷平台找灵感，选题随机，缺少阶段性路径。
- **内容生产成本高**：有素材但不知道如何转成图文、脚本、标题、首评和发布建议。
- **平台适配弱**：公众号、小红书、抖音、视频号的表达机制不同，普通用户很难准确迁移。
- **发完不会复盘**：发布后只看播放量/阅读量，无法判断问题出在标题、开头、选题、素材、结构还是互动设计。

### 3. 问题定义

本产品要解决的问题是：

> 如何让普通 KOC 在没有团队、没有成熟方法论的情况下，通过 AI 获得一套可执行、可发布、可复盘的内容增长工作流？

GrowthMate 不是单点文案生成器，而是一个面向普通创作者的 AI 内容增长 Agent。它把用户模糊的人设、素材和卡点，转化为“账号诊断、7 天增长任务、首条内容生成、发布后复盘”的完整链路。

### 4. 个人洞察来源

本产品的设计不只来自通用 Prompt，而结合了两类真实经验：

- **内容创作者经验**：本人有小红书账号运营经验，积累过 2000+ 粉丝，理解普通创作者在定位、素材包装、评论互动和内容持续性上的真实困难。
- **评测 PM 经验**：本人有评测 PM 实习经历，因此在产品设计中引入了评测集、LLM Judge、证据解释、Prompt 迭代等方法，而不是只停留在“生成一个看起来不错的结果”。

这也是 GrowthMate 与同质化“AI 文案生成器”拉开差异的关键：它不仅生成内容，也评估内容，并用评测结果反向改进 Agent。

## 模块二：产品方案设计

### 1. 产品概述

GrowthMate 是一款面向普通 KOC 的 AI 内容增长 Agent。

用户通过低压力表单输入自己的身份、人设、目标用户、平台、内容方向、已有素材和当前卡点后，系统会进入 Agent 工作流，完成：

1. 账号诊断
2. 7 天增长任务规划
3. 当前 selectedTopic 的完整内容生成
4. 发布建议
5. 基于真实发布稿和数据的复盘
6. LLM Judge 评测与 Prompt 迭代

产品核心价值是：把“我想做账号，但不知道怎么开始”转化为“今天可以发什么、用什么素材、发完看什么、下一条怎么改”。

### 2. 核心流程

GrowthMate 的主链路为：

用户画像输入 → Agent 诊断 → 7 天增长任务 → 选择首条任务 → 生成平台内容 → 输入真实发布稿和数据 → 复盘归因 → 推荐下一条任务。

其中，GrowthMate 采用**分步式 Agent 工作流**：

- 第一阶段不一次性生成 7 天所有正文，而是先规划增长路径。
- 第二阶段只对用户当前选择的任务生成完整内容，避免一次性输出过载。
- 第三阶段允许用户输入自己真实改过的发布稿，因为用户大概率不会原样发布 AI 文案。
- 第四阶段基于真实发布稿、数据和评论样本做复盘，形成下一轮内容方向。

### 3. 核心功能

#### 用户画像采集

采集用户背景、目标用户、内容赛道、平台、账号阶段、已有素材、内容边界和变现目标。输入设计尽量低压力，让新手不需要先学专业运营术语。

#### Agent 诊断

Agent 根据用户画像判断当前账号阶段、主要增长阻塞点、可用素材、不建议动作和下一步优先级。这里强调“先诊断后生成”，避免直接给一堆随机标题。

#### 7 天增长任务

系统生成 7 天任务，并按“建立信任、验证方向、轻转化/系列延展”组织顺序。每条任务包含：

- 目标
- 选题
- 内容形式
- 可用素材
- 发布后观察信号
- 评论区预埋设计

#### 内容生成

用户选择某一天任务后，系统生成对应平台的完整内容。不同平台有不同结构：

- 小红书：封面标题、正文、配图计划、话题标签、首评互动
- 抖音/视频号：开头钩子、分段口播、画面建议、字幕重点、评论触发
- 公众号：标题、导语、正文分节、插图建议、留言引导

#### 发布后复盘

用户发布后输入真实数据、典型评论和自己实际发布稿。Agent 会区分：

- 用户真实发布稿：复盘的一手证据
- Agent 建议稿：只能作为对比基准

系统会判断问题更可能出在标题、开头、选题、素材、内容结构、互动设计还是账号定位，并推荐下一条内容任务。

#### 评测后台

GrowthMate 内置 Eval Console，用于产品迭代和 Demo 展示。它包括：

- 固定评测集管理
- LLM Judge 评测结果
- 评测批次看板
- 低分维度、主要问题和 Prompt 建议
- 快照重评能力
- 线上用户回流的未来入口

### 4. 创新与差异化

#### 差异点一：从“文案生成”升级到“增长 Agent”

市面上大量同质化产品停留在标题、脚本、文案生成。GrowthMate 的重点不是写一篇稿子，而是帮用户判断：

- 我现在适合做什么定位？
- 第一周应该按什么顺序发？
- 哪条内容承担建立信任，哪条内容验证收藏价值？
- 发完以后应该看什么信号？
- 下一条应该继续、改写、扩展还是换方向？

#### 差异点二：结合真实 KOC 经验

普通创作者最大的阻力不是“不会写”，而是“不知道自己已有素材能不能用”“不知道平台用户为什么会评论/收藏”“不知道发完以后如何判断”。产品中特别强调：

- 绑定真实素材
- 避免装专家
- 不强行做爆款承诺
- 设计评论区反馈
- 低成本执行

这些设计来自真实内容创作经验，而不是单纯让 AI 罗列运营术语。

#### 差异点三：评测平台作为核心能力

GrowthMate 的亮点不是“用 AI 生成结果”，而是形成了一个评测驱动迭代闭环：

1. 构建 5 个固定评测 case，覆盖不同平台、赛道和用户阶段。
2. 用 LLM Judge 对 Agent 输出进行分维度评分。
3. Judge 输出分数、理由、证据、问题和 Prompt 迭代建议。
4. 将评测问题反向写回 Agent Prompt 和规范化逻辑。
5. 通过快照重评区分“Agent 输出变化”和“Judge 标准变化”。

这一点也更符合评测 PM 的工作方法：不是相信单次 AI 输出，而是建立样本、标准、评审、误判纠偏和版本对比机制。

#### 差异点四：快照重评机制

调 Judge Prompt 时不应该每次重新生成 Agent 输出，否则无法判断变化来自哪里。GrowthMate 支持读取历史 report 中的 generatedOutput，只重新运行 LLM Judge。这可以：

- 降低评测成本
- 避免重复调用生成链路
- 保证对比样本一致
- 支持评测标准持续迭代

## 模块三：AI 原生能力说明

### 1. 使用的 AI 能力

GrowthMate 使用 Kimi 模型完成以下 AI 能力：

- 多字段用户画像理解
- Agent 诊断和策略决策
- 7 天增长任务规划
- 平台化内容生成
- 真实发布稿复盘归因
- LLM-as-Judge 输出质量评测
- Prompt 迭代建议生成

### 2. AI 如何解决用户痛点

对于定位模糊，AI 根据用户身份、素材、目标人群和账号阶段，生成更具体的账号切口。

对于选题低效，AI 不只给标题，而是按阶段组织 7 天任务，让用户知道为什么今天发这条、发完看什么信号。

对于内容生产成本高，AI 把单条任务转化为平台可发布内容，包括正文/脚本、素材建议和互动设计。

对于平台适配弱，Prompt 中区分不同平台机制，例如小红书强调收藏和真实证据，抖音强调前 3 秒和完播，公众号强调导语承诺和读完率。

对于发布后不会复盘，AI 基于用户真实发布稿、数据和评论样本做证据驱动归因，并推荐下一条任务。

### 3. 技术方案

当前 Demo 使用轻量架构：

- 前端：HTML、CSS、JavaScript
- 后端：Node.js
- 模型：Kimi API，默认 `kimi-k2.6`
- 数据：本地 JSON/JSONL 文件
- Agent 接口：
  - `/api/generate`：诊断和 7 天任务规划
  - `/api/content`：当前任务内容生成
  - `/api/review`：发布后复盘
  - `/api/eval/*`：评测后台数据读取
- 评测脚本：
  - `npm run eval:judge`：运行 Agent + LLM Judge
  - `node scripts/run-eval.js --judge-report=...`：复用历史输出重评

### 4. Prompt 与结构化输出

系统通过 JSON 结构约束模型输出，保证前端可稳定渲染。Prompt 设计中加入：

- 先诊断后生成
- 平台指标校验
- 冷启动指标校准
- 关键记忆点覆盖
- 评论区预埋
- 用户真实发布稿和 Agent 建议稿边界
- 不伪造平台数据、不承诺涨粉

## 模块四：落地可行性

### 1. 当前完成情况

当前 Demo 已完成：

- 用户画像采集
- Agent 诊断
- 7 天任务规划
- 单条内容生成
- 发布后复盘
- PDF/方案导出能力
- Eval Console 评测后台
- 固定评测集
- LLM Judge
- 快照重评
- Prompt 迭代案例

### 2. 已验证的评测迭代案例

在敏感肌小红书 case 中，早期评测发现：

- 小红书 note 正文缺失
- 关键记忆点未被自然覆盖
- 复盘时容易混淆用户真实发布稿和 Agent 建议稿

迭代后：

- 内容生成补齐完整小红书正文
- mustMention 被传入内容生成和复盘
- evidence 开始标注来源，如【实际发布稿】、【发布数据】、【评论样本】、【Agent建议稿】
- 该 case 的 LLM Judge 结果达到 `85 / A`

### 3. 风险与挑战

- 模型生成延迟较高，评测运行需要支持异步任务队列。
- LLM Judge 自身也需要校准，避免用错误产品预期惩罚 Agent。
- 当前数据主要来自固定评测集，未来需要真实用户数据回流。
- 多用户版本需要账号体系、数据库和历史方案管理。

### 4. 后续计划

- 页面内一键运行评测和异步进度展示
- 人工标注队列，用人工复核校准 LLM Judge
- Prompt 版本管理，追踪每次迭代对不同 case 的影响
- 接入真实用户回流数据，比较用户反馈与模型评审的一致性

## 模块五：商业化思考

### 1. 目标市场

GrowthMate 面向普通 KOC、低粉创作者、个人品牌建设者、MCN 新人培训和品牌素人营销场景。

### 2. 盈利模式

- 个人订阅：按月使用诊断、内容生成和复盘功能。
- 按次付费：按生成方案、内容和评测次数计费。
- 创作者工具包：提供选题库、复盘模板、评论区模板和脚本模板。
- B 端 SaaS：为 MCN 或品牌方提供批量 KOC 起号方案、Brief 生成和内容复盘。
- 平台合作：与视频号、公众号、创作者工具生态结合。

### 3. 竞争优势

- 面向普通 KOC，而不是专业团队。
- 链路完整，覆盖诊断、规划、生成、复盘和评测。
- 结合真实内容创作者经验，强调低成本、素材绑定和评论反馈。
- 引入评测 PM 方法论，构建固定评测集、LLM Judge 和快照重评。
- 可持续迭代，不依赖一次性 Prompt 效果。

## 总结

GrowthMate 的核心创新不是“让 AI 写内容”，而是把 AI 用在普通 KOC 的完整增长流程中，并通过评测后台持续改进 Agent 质量。

它把创作者经验、AI Agent 工作流和评测 PM 方法结合起来，形成了一个从生成到评测再到迭代的闭环。这使它在同类 AI 内容生成产品中具备更强的差异化和可讲述性。
